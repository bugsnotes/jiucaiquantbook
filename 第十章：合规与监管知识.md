# 第九章：合规与监管知识

量化交易不仅需要技术能力,更要了解法律法规和监管要求。本章介绍个人和机构量化交易需要注意的合规事项。

## 9.2 个人量化交易合规要点

### 合法交易行为

#### 允许的操作
```
✓ 使用个人账户程序化交易
✓ 使用公开数据进行分析
✓ 编写自动交易程序
✓ 使用券商提供的API
✓ 参与量化基金
✓ 购买合法数据服务
```

#### 禁止的操作
```
✗ 使用他人账户
✗ 内幕交易
✗ 操纵市场
✗ 传播虚假信息
✗ 违规获取信息
✗ 恶意做空
```

### 内幕交易

**定义**:
```
利用未公开重大信息进行交易

构成要素:
1. 内幕信息
2. 知情人员
3. 交易行为
4. 获利或避损
```

**案例警示**:
```
❌ 案例1: 上市公司员工
- 提前知道财报数据
- 财报公布前买入
- 构成内幕交易

❌ 案例2: 爬虫获取未公开信息
- 爬取公司内部系统
- 提前知道业绩
- 交易获利
- 构成违法

⚠️ 即使是量化策略,如果使用内幕信息,也违法!
```

**如何避免**:
```
✓ 只使用公开信息
✓ 避免接触内幕信息渠道
✓ 建立信息隔离
✓ 记录信息来源
```

### 市场操纵

**常见形式**:
```
1. 连续交易操纵
   - 连续买卖拉抬股价
   - 制造虚假繁荣

2. 约定交易
   - 与他人串通
   - 影响价格

3. 洗售
   - 自买自卖
   - 制造成交假象

4. 虚假申报
   - 大量申报后撤单
   - 影响市场预期

5. 尾盘拉抬/打压
   - 收盘前大单
   - 影响收盘价
```

**量化中的风险**:
```
⚠️ 风险1: 高频刷单
- 频繁报撤单
- 可能被认定为虚假申报

⚠️ 风险2: 尾盘交易
- 算法在尾盘集中交易
- 可能影响收盘价

⚠️ 风险3: 对敲
- 多账户策略
- 可能构成约定交易

解决:
✓ 合理控制频率
✓ 分散交易时间
✓ 避免多账户
✓ 记录交易逻辑
```

### 异常交易监控

**交易所监控指标**:
```
1. 日内回转交易
   - T+0交易(不允许)
   - 监控同一只股票日内买卖

2. 虚假申报
   - 大量撤单
   - 撤单率过高

3. 拉抬打压
   - 大额买卖
   - 价格异常波动

4. 频繁交易
   - 交易频率过高
   - 超过正常水平
```

**个人如何应对**:
```
1. 了解规则
   - 熟悉交易所规定
   - 知道红线在哪

2. 合理交易
   - 避免极端行为
   - 保持正常频率

3. 保留证据
   - 策略逻辑文档
   - 交易记录
   - 可解释性

4. 响应监管
   - 及时回复交易所问询
   - 如实说明情况
```

### 账户管理

**账户规范**:
```
✓ 使用本人实名账户
✓ 保护账户安全
✓ 不出借账户
✓ 不使用他人账户

❌ 违规行为:
- 出借账户给他人使用
- 代客理财(无资质)
- 账户拆借
- 虚拟账户
```

**多账户问题**:
```
问题: 个人拥有多个账户是否合规?

答案:
- 个人可以开多个账户(不同券商)
- 但不能用于操纵市场
- 不能协同交易影响价格
- 量化策略需要特别注意

建议:
- 个人尽量使用单账户
- 如需多账户,确保独立策略
- 避免同时交易同一股票
```

## 9.3 机构量化合规

### 私募基金管理人

#### 备案要求
```
1. 登记为私募基金管理人
   - 向中国证券投资基金业协会登记
   - 提交材料
   - 通过审核

2. 高管资格
   - 基金从业资格
   - 无不良记录
   - 专业能力

3. 注册资本
   - 实缴资本要求(通常1000万+)

4. 办公场所
   - 实际经营场所
   - 与经营规模匹配
```

#### 产品备案
```
流程:
1. 完成管理人登记
2. 产品设计
3. 向协会备案
4. 备案通过后募集

要求:
- 合格投资者(100万+)
- 不超过200人
- 不公开宣传
- 不承诺收益
```

#### 信息披露
```
定期披露:
- 季度报告
- 年度报告
- 重大事项报告

内容:
- 净值
- 持仓(可简化)
- 业绩
- 风险
```

### 公募基金

#### 设立门槛
```
要求:
- 注册资本1亿+
- 股东资质严格
- 高管团队专业
- 完善制度
- 监管审批

现实:
- 个人几乎不可能
- 机构才能申请
```

#### 投资限制
```
单一股票:
- 不超过基金资产10%
- 不超过上市公司流通股10%

投资范围:
- 限定在招募说明书范围内
- 不能偏离投资目标

杠杆限制:
- 一般不超过1倍
- 部分基金可融资

影响量化:
- 单股票仓位限制
- 对高集中度策略有影响
```

### 券商资管

```
1. 集合资管
   - 类似公募
   - 多个客户
   - 统一管理

2. 定向资管
   - 一对一
   - 定制化

3. 专项资管
   - 特定目的
```

**合规要点**:
```
- 券商资管牌照
```

## 9.4 程序化交易监管

### 程序化交易定义

**官方定义**:
```
通过计算机程序自动生成或执行交易指令
进行证券交易的行为

包括:
- 自动下单
- 算法交易
- 高频交易
- 量化策略
```

### 报备制度

**谁需要报备**:
```
1. 机构投资者
   - 私募基金
   - 资管产品
   - 自营业务

2. 达到一定规模的个人
   - 日均交易金额超过阈值
   - 交易频率过高
   (具体标准由交易所确定)

3. 券商
   - 为客户提供程序化交易接口
   - 需要向交易所报备
```

**报备内容**:
```
- 策略类型
- 交易品种
- 预计交易规模
- 风控措施
- 技术系统
- 负责人信息
```

**个人是否需要报备**:
```
目前(2024):
- 大部分个人不需要报备
- 但需要通过券商报备的系统交易

未来可能:
- 监管趋严
- 可能要求更多报备

建议:
- 关注监管动态
- 咨询券商要求
- 保持合规
```

### 高频交易限制

**定义**:
```
高频交易(HFT):
- 极短持仓时间(秒级、毫秒级)
- 极高交易频率
- 极小单笔盈利
- 大量订单

监管关注:
- 可能影响市场稳定
- 占用交易资源
- 对普通投资者不公平
```

**限制措施**:
```
1. 最小报单间隔
   - 上交所: 500毫秒
   - 深交所: 500毫秒
   (避免瞬间大量报单)

2. 撤单率限制
   - 撤单次数/成交次数 < 一定比例
   - 过高会被限制

3. 异常交易监控
   - 频繁报撤单
   - 价格异常波动
   - 大额交易

4. 风控要求
   - 订单流量控制
   - 价格偏离限制
   - 熔断机制
```

**对个人的影响**:
```
一般个人量化:
- 日线/分钟线策略不受影响
- 正常交易频率无问题

高频个人:
- 受到限制
- 难以开展真正高频
- 需要专业系统

建议:
- 个人避免高频
- 专注中低频策略
```

**合法数据源**:
```
✓ 公开市场数据
✓ 交易所官方数据
✓ 购买的合法数据服务
✓ 公司公告
✓ 新闻媒体
```

**非法数据源**:
```
✗ 未经授权的系统数据
✗ 爬取禁止爬取的网站
✗ 窃取的数据
✗ 内幕信息
✗ 侵犯隐私的数据
```

### 爬虫法律风险

**合法爬虫**:
```
条件:
1. 遵守robots.txt
2. 合理请求频率
3. 公开可访问信息
4. 不侵犯著作权
5. 不侵犯隐私

示例:
- 爬取公开新闻
- 爬取公告
- 合理范围内的公开数据
```

**非法爬虫**:
```
风险行为:
✗ 爬取需要登录的内部数据
✗ 破解反爬机制
✗ 过度请求导致服务中断
✗ 爬取个人隐私信息
✗ 用于商业牟利(未授权)

法律后果:
- 民事赔偿
- 行政处罚
- 刑事责任(严重情况)
```

**案例**:
```
某量化团队爬取案例:
- 爬取电商销售数据
- 未经授权
- 用于预测上市公司业绩
- 交易获利
- 被判定违法

教训:
- 使用合法数据源
- 购买授权数据
- 不碰法律红线
```

### 个人信息保护

**《个人信息保护法》**:
```
影响量化:
- 不能随意收集个人信息
- 舆情数据需要脱敏

建议:
- 使用公开的匿名数据
```

## 9.7 税务合规

### 个人投资税务

**资本利得税**:
```
当前政策(2024):
- 个人股票交易暂免征收资本利得税
- 只需缴纳印花税

印花税:
- 卖出时征收: 成交额 × 0.1%
- 买入免征

注意:
- 未来可能开征资本利得税
- 关注政策变化
```

**分红税**:
```
股息红利税:
- 持股<1个月: 20%
- 持股1个月-1年: 10%
- 持股>1年: 5%

影响:
- 频繁交易的量化策略
- 收到分红缴税较高

优化:
- 分红前卖出(如果策略允许)
- 或长期持有降低税率
```

### 机构税务

**企业所得税**:
```
税率: 25%

计税:
- 投资收益计入应纳税所得
- 扣除成本费用
- 按25%缴税

私募基金:
- 合伙企业形式常见
- 税务穿透到投资者
```

**增值税**:
```
金融服务增值税:
- 资管产品管理费: 6%
- 金融商品转让: 免征(部分情况)

影响:
- 私募管理费需要缴纳增值税
```

## 9.8 风险防范

### 技术风险

**系统风险**:
```
风险:
- 程序错误
- 网络中断
- 服务器宕机
- 数据错误

防范:
✓ 充分测试
✓ 异常处理
✓ 备份系统
✓ 人工监控
✓ 熔断机制
```

**风控措施**:
```python
# 示例: 风控检查
def risk_check(order):
    """下单前风控检查"""

    # 1. 单笔金额检查
    if order['amount'] > MAX_ORDER_AMOUNT:
        raise Exception("单笔金额超限")

    # 2. 单日交易次数
    if daily_order_count > MAX_DAILY_ORDERS:
        raise Exception("单日交易次数超限")

    # 3. 价格偏离检查
    if abs(order['price'] - current_price) / current_price > 0.05:
        raise Exception("价格偏离过大")

    # 4. 持仓检查
    if total_position > MAX_POSITION:
        raise Exception("总仓位超限")

    return True
```

### 操作风险

**常见错误**:
```
1. 胖手指(Fat Finger)
   - 输入错误数量
   - 例如: 100股 误输成 100000股

   防范:
   - 订单确认
   - 金额上限
   - 二次确认

2. 重复下单
   - 网络延迟导致重复点击

   防范:
   - 防重复机制
   - 订单去重

3. 账户混淆
   - 实盘账户/模拟账户搞混

   防范:
   - 明确标识
   - 环境隔离
```

### 合规风险

**定期自查**:
```
检查清单:
□ 是否使用内幕信息
□ 是否操纵市场
□ 交易频率是否异常
□ 数据来源是否合法
□ 账户使用是否规范
□ 是否及时纳税
□ 信息披露是否完整(机构)
```

**应对检查**:
```
如果收到监管部门或交易所问询:

1. 及时响应
   - 不要拖延
   - 在规定时间内回复

2. 如实说明
   - 提供交易逻辑
   - 提供策略说明
   - 提供相关证据

3. 寻求专业帮助
   - 咨询律师
   - 咨询券商
   - 必要时聘请专业机构

4. 整改
   - 如果确有不当
   - 立即整改
   - 避免再犯
```

## 本章小结

本章介绍了量化交易的合规与监管知识:

### 核心要点

**1. 监管框架**:
